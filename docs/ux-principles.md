# UX Principles

## 1. Native over imitated
The app should behave like a great Mac app, not like a browser product mimicking one.

## 2. Calm power
The interface should feel capable and rich without becoming visually noisy or overwhelming.

## 3. Keyboard first, mouse friendly
A power user should be able to navigate, edit, invoke commands, and use AI efficiently from the keyboard.

## 4. File trust is sacred
Users should always understand where their documents live and trust that the app respects their files.

## 5. Raw and rendered should cooperate
Markdown syntax and polished reading presentation should not feel like separate modes at war with each other.

## 6. AI should feel available, not intrusive
AI should be easy to invoke when helpful and easy to ignore when not.

## 7. Fast by feel, not only by benchmark
The user should perceive the app as immediate, responsive, and lightweight.

## 8. Progressive depth
The basics should be obvious, while more advanced capabilities reveal themselves naturally.

## 9. System alignment
Use native patterns for settings, menus, windows, drag and drop, search, focus, and previews whenever practical.

## 10. Respect attention
Avoid unnecessary animation, clutter, or interruptions that break writing flow.

## Core interaction goals

### Workspace navigation
- clear file tree
- predictable selection behavior
- easy recent workspace reopening
- responsive search and jump-to-file flows

### Editing
- minimal friction from open to edit
- clear visual hierarchy in markdown documents
- smooth cursor movement and selection
- intuitive behavior around headings, lists, links, code blocks, and block transforms

### AI
- easy selected-text actions
- contextual assistance rather than generic chat only
- visible model/task status
- simple cancellation and recovery

### Secondary content
- PDFs should preview quickly and naturally
- dictation should feel native and privacy-conscious
- previews should not hijack the writing flow
