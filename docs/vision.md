# Vision

## Product vision

Cushion for Mac will be the best native markdown workspace for Apple silicon users who want the flexibility of plain files, the power of AI assistance, and the speed and feel of a serious Mac application.

It should feel like it belongs alongside the best Mac productivity tools rather than like a website trapped in a window.

## Product thesis

The existing Cushion experience proves there is value in combining markdown-centric thinking, local-first trust, and embedded AI support. The Mac rewrite should preserve that value while replacing the interaction model, architecture, and performance profile with something that feels purpose-built for macOS.

## What we are building

A native macOS application that helps users:

- browse and organize local markdown workspaces
- write and edit markdown fluidly
- move between raw syntax and polished presentation without friction
- use AI to summarize, rewrite, explore, and act on their notes
- preview and work with PDFs and adjacent file types inside the workspace
- optionally use dictation and other Mac-native intelligence features
- benefit from fast startup, smooth editing, low memory usage, and strong system integration

## Who it is for

Primary users:

- knowledge workers who live in markdown
- developers and technical thinkers who prefer local files
- researchers, writers, and note-heavy operators
- Mac users who care about polish, keyboard speed, and native performance

Secondary users:

- AI-curious users who want assistance within a document workspace
- users leaving Electron-based note tools because they want lower memory use and better system feel

## What makes it special

- plain markdown files remain portable and trustworthy
- the app is local-first by default
- the writing experience is native and fast
- AI is integrated without owning the entire product
- the app leverages Apple silicon and modern macOS frameworks intelligently

## Experience goals

The app should feel:

- immediate
- calm
- keyboard-friendly
- trustworthy
- powerful without being noisy
- unmistakably native

## Strategic decisions

### Native over wrapped
We are explicitly choosing a native rewrite rather than a wrapped web app.

### File-first over database-first
User content remains in portable markdown files.

### Editor-first over AI-first
The editor must be excellent before AI becomes a differentiator.

### Modular over monolithic
The architecture should support future expansion without collapsing under complexity.

### Apple-native over novelty-chasing
Adopt state-of-the-art Apple technologies where they materially improve the product, not to collect buzzwords.

## V1 product promise

A user can open a markdown workspace on Mac, navigate it fluidly, edit files in a beautiful high-performance native editor, access helpful AI assistance, preview relevant attachments such as PDFs, and trust the app to behave like a high-quality native Mac tool.
