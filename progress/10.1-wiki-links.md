# Phase 10.1 — Wiki-Style Links

## Status: ✅ Complete

## Overview

Implemented wiki-style links (`[[note]]` syntax) inspired by Obsidian and Tangent. Links are displayed WYSIWYG-style with brackets hidden when the cursor is not on the line.

## Features

### Syntax Support
- `[[note]]` — Basic link to another file
- `[[folder/note]]` — Link with path
- `[[note#header]]` — Link with header anchor
- `[[note|display text]]` — Link with custom display text
- `[[note#header|display]]` — Full syntax with all options

### Link Resolution
- **Fuzzy matching**: Case-insensitive, partial paths, filename-only matching
- **Extension inference**: Prefers `.md` files when no extension specified
- **Three states**:
  - `resolved` — Target file found (styled blue)
  - `empty` — No matching file (styled red with dashed underline)
  - `ambiguous` — Multiple matches (styled purple with dotted underline)

### WYSIWYG Display
- Brackets `[[` and `]]` hidden when cursor is not on the line
- Brackets revealed in faint color when editing the line
- Content anchor (`#header`) hidden in display mode
- Display text shown instead of full path when using `|` syntax

### Navigation
- **Click** to navigate to linked file (like Obsidian)
- If link target doesn't exist, creates new file with:
  - Path based on href (adds `.md` extension if needed)
  - Initial content with H1 title from the href
- File browser refreshes to show newly created files

## Regex Pattern

```typescript
// Adapted from Tangent's wikiLinkMatcher
const wikiLinkRegex = /\[\[([^\[\]|#\n]+)(#[^\[\]|#\n]*)?(\|[^\[\]\n]*)?\]\]/g;

// Groups:
// [1] href      - "My Note" or "folder/My Note"
// [2] contentId - "#header" (optional)  
// [3] text      - "|display text" (optional)
```
