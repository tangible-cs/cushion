- Inputs: OpenCode message rendering + streaming behavior, Cushion layout patterns
- Outputs: Guardrails, metrics, and risks (no code).
- Notes:
  - Rendering guardrails:
    - Use incremental turn rendering (initial batch + backfill) to avoid large DOM spikes.
    - Throttle streaming text render updates to avoid markdown reparse churn.
  - Scroll performance:
    - Keep a single scroll container in the sidebar; avoid nested scroll regions in message list.
    - Maintain bottom padding equal to prompt height to prevent layout thrash.
  - State updates:
    - Batch SSE updates by message/part id to minimize store writes.
    - Avoid re-sorting message arrays on every update; insert by id.
  - Media handling:
    - Lazy render large tool outputs and diff previews; collapse by default where possible.
  - Metrics to track manually:
    - Time to first message render after send.
    - Stream update smoothness (no visible flicker).
    - Scroll jank when history backfills.
- Decisions:
  - Prioritize streaming throttle + turn backfill in MVP before adding diff-heavy tool parts.
- Rule: Copy from OpenCode when it fits perfectly; avoid unnecessary implementation.
