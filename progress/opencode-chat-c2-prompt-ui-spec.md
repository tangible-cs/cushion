# C2 - Prompt UI Spec

- Status: [x] Complete
- Owner: Subagent
- Scope: Cushion-native UI layout and interaction rules.
- Inputs: Cushion layout (`apps/frontend/app/page.tsx`), OpenCode prompt behaviors, existing Cushion sidebar patterns
- Outputs: UI spec and interaction notes (no code).
- Notes:
  - Placement: Chat lives in the right sidebar and must coexist with Backlinks/Graph/Terminal. Use a tabbed or stacked panel that preserves width and avoids layout reflow.
  - Structure (top-to-bottom):
    - Context pills row (file selections, comments) with remove action and click-to-focus behaviors.
    - Attachment strip for images/PDFs with thumbnails and remove control.
    - Contenteditable input area with monospace fallback in shell mode.
    - Footer row for agent/model selection, attach button, context-usage indicator, and send/stop control.
  - Visual rules:
    - Use Cushion typography/spacing tokens; avoid introducing new font families.
    - Keep the input container rounded and elevated to match existing surface styling.
    - Keep popovers (slash/@) inside the sidebar width; default to max-height with scrolling.
  - Interaction rules:
    - Send button toggles to stop while streaming; tooltip shows Enter vs Esc.
    - Slash and @ popovers use keyboard navigation with Tab to select; Escape closes.
    - Shift+Enter inserts newline; Enter submits when not composing IME.
    - Shell mode shows a mode badge with exit hint; use distinct styling (monospace + subtle highlight).
  - Accessibility rules:
    - Contenteditable has proper aria-label; placeholder text is visually shown only when empty.
    - Buttons expose aria-labels and consistent focus rings.
  - State coupling:
    - Prompt UI must react to global session status (busy/idle) for send/stop state.
    - Context pill clicks must open the relevant file or review panel without leaving the sidebar.
- Decisions:
  - Use a single chat footer row to avoid sidebar height overflow; overflow sections scroll within the input container.
- Rule: Copy from OpenCode when it fits perfectly; avoid unnecessary implementation.
