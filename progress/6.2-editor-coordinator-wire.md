# 6.2 Editor-Coordinator Wire

## Status
COMPLETED

## Summary
Wiring done inside EditorPanel.tsx: sends textDocument/didOpen on first render, textDocument/didChange on every edit, and workspace/save-file on Ctrl+S.

## Description
Wire the editor to the coordinator: send `textDocument/didOpen` when a file is opened, `textDocument/didChange` on every edit, and trigger save-file on save (Ctrl+S).

## Files Involved
| File | Role |
|---|---|
| `apps/frontend/src/components/editor/EditorPanel.tsx` | Editor component with coordinator wiring |
| `apps/frontend/src/lib/coordinator-client.ts` | Send RPC calls |
| `apps/coordinator/src/server.ts` | Handle didOpen, didChange, save |

## Acceptance Criteria
- [x] Opening a file sends `textDocument/didOpen` to coordinator
- [x] Every edit sends `textDocument/didChange` with the changes
- [x] Ctrl+S triggers a save-file request to coordinator
- [x] Coordinator writes the file to disk on save
- [x] Editor reflects the saved state (no unsaved indicator after save)
- [x] Error states are handled (file not found, write failure)

## Notes
- Consider debouncing didChange notifications for performance.
