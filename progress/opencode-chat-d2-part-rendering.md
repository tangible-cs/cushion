# D2 - Part Rendering Rules

- Status: [x] Complete
- Owner: Subagent
- Scope: Render rules for text/file/tool/diff/terminal parts.
- Inputs: `opencode/packages/ui/src/components/session-turn.tsx`, `opencode/packages/ui/src/components/message-part.tsx`
- Outputs: Rendering matrix and edge cases (no code).
- Notes:
  - Rendering is split by message role:
    - User messages render a summary view: attachments (images/PDFs), text with highlights, and inline references to file/agent parts.
    - Assistant messages render each part via a part registry.
  - User message rendering:
    - Text is the first non-synthetic text part; includes copy action and expandable view if long.
    - File/agent references are highlighted using source start/end offsets to mark inline mention spans.
    - Attachments are displayed as thumbnails or file icons; images open a preview modal on click.
  - Assistant message rendering:
    - Tool parts render via a tool registry; includes status, error display, output sections, and permission/question prompts when gated.
    - Text parts render markdown; include copy action.
    - Reasoning parts render markdown separately and can be hidden by preference.
    - File parts are used both as attachments and inline references; image/PDF file parts are treated as attachments.
  - SessionTurn logic:
    - User message drives the turn; assistant messages with parentID are grouped under the user message.
    - Attachments are split from inline parts to support sticky layout.
    - Status text is derived from the most recent tool/reasoning/text part and used for a running status indicator.
  - Tool render variants:
    - Specific tool names map to icons/titles and custom layouts; unknown tools fall back to a generic renderer.
    - Errors render prominently with structured title/body.
- Decisions:
  - For Cushion, implement a minimal part registry: text, tool, file (attachment), reasoning; expand as needed.
- Rule: Copy from OpenCode when it fits perfectly; avoid unnecessary implementation.
