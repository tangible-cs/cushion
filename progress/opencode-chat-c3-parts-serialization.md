# C3 - Prompt to Parts Serialization

- Status: [x] Complete
- Owner: Subagent
- Scope: Define how prompt content becomes message parts.
- Inputs: `opencode/packages/app/src/components/prompt-input.tsx`, prompt context definitions
- Outputs: Serialization rules and constraints (no code).
- Notes:
  - Base message text is built by concatenating prompt parts that carry content (text, file pills, agent pills) in their DOM order.
  - File attachment parts:
    - Each file pill becomes a file part with mime text/plain and a file:// URL.
    - URL includes an optional ?start=LINE&end=LINE query if a selection exists.
    - Source metadata includes extracted text plus start/end offsets and absolute path.
  - Agent parts:
    - Each agent pill becomes an agent part with name and source text offsets.
  - Context parts:
    - Each context item becomes a file part with a file:// URL.
    - If the context item includes a comment, a synthetic text part is prepended that describes the comment and the selected range.
    - Duplicate file URLs are de-duplicated unless a comment is present.
  - Image attachments:
    - Each image becomes a file part with mime set to the image mime and url set to the data URL.
  - Ordering:
    - Request parts are ordered as: text part, file attachments, context parts, agent parts, image parts.
  - Session routing:
    - For new sessions, worktree selection may change the directory context; prompt is sent via a client scoped to that directory.
- Decisions:
  - Preserve the same serialization order and synthetic comment text behavior in Cushion to match OpenCode expectations.
- Rule: Copy from OpenCode when it fits perfectly; avoid unnecessary implementation.
